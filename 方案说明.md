# JOJOUP 初中达人筛选工具方案

## 1. 页面规划

这个工具建议做成「筛选策略 + 达人榜单 + 自动建议 + 详情溯源」的工作台，而不是单纯表格。

核心页面结构：

1. 顶部：项目名称、数据更新时间、导入名单、导出清单、重新评分。
2. 数据源状态区：星图达人广场、星图人群包匹配、抖音公开视频数据、竞品/内容标注四类数据是否已接入。
3. 左侧筛选策略：目标人群、适配年级、报价上限、最低推荐分、五维权重、是否隐藏高竞品饱和账号。
4. 中间达人候选池：展示达人ID、抖音链接/星图ID、推荐分、五维表现、人群/内容标签、报价效率、竞品状态、合作建议。
5. 右侧自动建议：输出A档优先邀约、B档测试合作、C档观察补数；点击达人后展示推荐理由、Brief切角和数据缺口。
6. 后续可扩展二级页：达人对比、投后复盘、竞品监测、评分规则管理。

本目录里的 `index.html` 已经按这个结构做了一个可交互原型，可直接用浏览器打开。

## 1.1 对抗性审查合并结论

根据子代理对方案的独立审查，工具必须遵守以下约束，否则会把“看起来有数据”误判成“可以放量”：

1. 推荐分和置信度必须拆开：推荐分回答值不值得投，置信度回答证据够不够。低置信度账号不能进A档，只能进入补数或小预算测试。
2. 每个关键指标必须尽量带上 `数据来源`、`权限状态`、`数据周期`、`更新时间`、`样本作品数`。星图导出、星图人群包、抖音授权、人工标注、投后回流不能混为一谈。
3. 竞品投放不是单纯正负项。投过教育类说明商业承接可能存在，但近期高频同类APP合作意味着饱和和排他风险，应直接降档。
4. A3、看后搜、组件点击、应用下载等投后指标只能在已有合作或平台授权后作为强信号；前选阶段不能假设任意达人都有这些数据。
5. 学生向内容必须单独做未成年人和脚本合规审查；涉及承诺提分、升学保障、官方/命题/阅卷暗示、焦虑营销时要先暂缓。
6. 传播表现不能只看粉丝和爆款，应优先看近30/60天稳定播放、互动质量、样本量和商单视频表现。

这些约束已经在 `index.html` 里落成：CSV导入、导出、置信度惩罚、数据缺口、合规提示、低置信不进A档。

## 2. 五维评分口径

截图里的五个维度可以落成如下字段和评分：

| 维度 | 建议权重 | 关键字段 | 评分逻辑 |
| --- | ---: | --- | --- |
| 近期传播数据表现 | 22% | 近30/60日均播、互动率、完播率、粉丝增速、CPM/CPE | 不只看爆款，重点看稳定均播和互动质量 |
| 人群画像精准度 | 24% | 家长/学生占比、年龄、地域、教育兴趣、初中相关内容占比、人群包匹配度 | JOJOUP优先看初中生和30-45岁家长触达 |
| 商业能力表现 | 22% | 星图商单经验、A3新增、看后搜/回搜、组件点击、履约记录 | 用历史商单和投后数据判断能否有效种草 |
| 竞品投放但非饱和 | 17% | 近90天竞品合作次数、最近合作日期、合作行业、是否排他 | 投过教育类是正信号，高频同类APP投放要扣分 |
| 内容切角 | 15% | 家长向/孩子向/混合、学科、年级、内容形式、脚本可控度 | 匹配JOJOUP全科APP卖点，避免只绑定单科或泛鸡汤 |

推荐分阈值：

| 分数 | 建议 |
| --- | --- |
| 86+ | A档，优先邀约和建联 |
| 78-85 | B档，低成本测试或作为矩阵补充 |
| 70-77 | C档，先补数据或观察排期 |
| 70以下 | 暂不推荐 |

## 3. 数据来源与接入方式

### P0：先用官方导出 + CSV导入跑通

第一版不要直接爬抖音或星图后台。建议先用星图官方导出能力和人工补录跑通评分模型：

1. 在巨量星图达人广场按教育、亲子、初中、学科、报价、粉丝量、商单经验等条件筛选。
2. 使用星图「导出数据」下载达人名单，名单包含筛选项、昵称、星图ID等达人信息。
3. 按 `数据字段模板.csv` 统一字段，并标记数据来源、权限状态、数据周期、更新时间和样本作品数，导入工具后自动计算五维分和置信度。
4. 抖音主页链接、近期公开视频表现、竞品合作标签可以先用人工/半自动表格补齐。

注意：星图导出功能依赖客户等级和平台权限，官方说明里提到客户等级需大于等于L3级才可使用导出能力。

### P1：接官方API和授权数据

数据接入优先级：

1. 巨量星图/巨量引擎商业开放平台：优先申请客户侧、代理商侧或服务商侧接口权限，拉取达人基础信息、订单、投后、线索、A3等数据。公开文档里已有星图相关API和抖音达人相关API类目，但具体字段要以账号开通权限为准。
2. 抖音开放平台：用于补齐公开视频维度。官方视频数据方案面向第三方open_api接入，适合拉取视频点赞、分享、评论、粉丝等数据，但需要抖音账号授权。
3. 星图投后报告：合作后把传播表现、搜索表现、人群资产、观众画像、组件点击、应用下载等投后指标回流，更新达人历史表现和下一轮评分。
4. 人群包找达人：在星图后台用初中生、家长、教育兴趣等TA包做人群匹配，拿目标人群覆盖度和达人观众匹配度。

不建议作为主路径：

1. 模拟登录星图或抖音网页抓取数据，稳定性和合规风险都高。
2. 用非官方第三方库抓取达人画像，教育产品投放需要更谨慎，数据来源不可解释会影响后续复盘。

## 4. 数据表设计

最少需要四张逻辑表：

| 表 | 主键 | 说明 |
| --- | --- | --- |
| creator_profile | creator_id | 达人ID、星图ID、抖音链接、昵称、粉丝数、报价、类目、MCN |
| creator_metrics | creator_id + date_range | 近30/60天播放、互动、完播、粉丝增长、CPM/CPE |
| audience_profile | creator_id + date_range | 年龄、性别、地域、教育兴趣、家长/学生倾向、人群包覆盖度 |
| creator_evaluation | creator_id + project_id | 五维分、推荐等级、竞品风险、内容切角、Brief建议、人工备注 |
| metric_lineage | creator_id + metric_name | 指标来源、权限状态、数据周期、更新时间、样本量、人工/官方标记 |
| compliance_review | creator_id + project_id | 合规风险词、未成年人风险、硬拦截原因、脚本审核状态 |

导入模板已放在 `数据字段模板.csv`。

## 5. 自动建议逻辑

工具应输出的不只是排名，还要给投放动作：

1. 首批邀约：推荐分高、人群匹配高、竞品未饱和、报价可控。
2. 低成本测试：传播强但商业样本少，适合先拿素材和小预算验证。
3. 节点型合作：如中考、期末、开学季匹配强，但报价高或竞品多，需要排期和排他确认。
4. 暂缓：竞品饱和、内容调性不符、学生向内容商业转化弱、历史商单数据不足。

## 6. 迭代路线

1. 第1版：CSV/飞书表格导入、五维评分、推荐清单导出。
2. 第2版：接星图导出解析、人群包匹配结果导入、竞品词半自动标注。
3. 第3版：接巨量/抖音官方API，自动更新近期数据和投后数据。
4. 第4版：加入达人对比、投后复盘、预算组合优化和Brief自动生成。
